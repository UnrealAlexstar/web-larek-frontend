# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Webpack

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- src/pages/index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/index.ts — точка входа приложения
- src/scss/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter):
- **Model (Слой данных)** — отвечает за хранение и изменение данных.
- **View (Слой отображения)** — отвечает за вывод данных на экран и отлов действий пользователя.
- **Presenter (Слой представления)** — связывает данные и отображение. В данном проекте роль Presenter выполняет код в основном файле `index.ts`, который слушает события через брокер событий и реагирует на них.

### Базовый код

#### Класс `Api`
Содержит в себе базовую логику отправки запросов. В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.
- `get` — выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер.
- `post` — принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса можно переопределить третьим параметром при вызове.

#### Класс `EventEmitter`
Брокер событий, позволяющий отправлять события и подписываться на них. Используется для связи слоев приложения (паттерн Observer).
- `on` — подписка на событие.
- `emit` — инициализация события.
- `trigger` — возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие.

#### Класс `Component<T>`
Абстрактный базовый класс для всех компонентов слоя отображения. Предоставляет инструментарий для работы с DOM.
- `toggleClass` — переключает класс элемента.
- `setText` — устанавливает текстовое содержимое.
- `setDisabled` — меняет статус блокировки элемента.
- `setHidden` / `setVisible` — скрывает или показывает элемент.
- `render` — основной метод, возвращающий корневой DOM-элемент компонента с обновленными данными.

### Слой данных (Model)

#### Класс `AppState`
Центральный класс, хранящий состояние всего приложения.
- Хранит каталог товаров, содержимое корзины, данные заказа и ошибки валидации форм.
- Методы: `setCatalog()`, `addToBasket()`, `removeFromBasket()`, `clearBasket()`, `setOrderField()`, `validateForm()`. При изменении данных вызывает соответствующие события через `EventEmitter`.

### Слой отображения (View)

Все классы отображения наследуются от базового класса `Component<T>`. Они не хранят бизнес-логику, а только отображают переданные в них данные и сообщают о действиях пользователя.

- **`Page`** — управляет главной страницей (отображение каталога, счетчик корзины, блокировка прокрутки при открытии модального окна).
- **`Modal`** — универсальный компонент модального окна. Умеет открываться, закрываться и принимать внутрь себя любую разметку (контент).
- **`Card`** — отображает карточку товара. В зависимости от переданного шаблона может выглядеть по-разному (в каталоге, в модальном окне или в корзине). Генерирует события клика по карточке или кнопке «Купить».
- **`Basket`** — отображает список товаров в корзине и общую стоимость. Генерирует событие перехода к оформлению заказа.
- **`Form`** — базовый класс для работы с формами. Управляет полями ввода, кнопкой сабмита и выводом ошибок валидации.
  - **`OrderForm`** — расширяет `Form`, отвечает за первый шаг оформления (адрес и способ оплаты).
  - **`ContactsForm`** — расширяет `Form`, отвечает за второй шаг (телефон и email).
